# 部署说明文档

## 1. 环境要求

### 1.1 开发环境
- **JDK**：17 或更高版本
- **Maven**：3.6 或更高版本
- **MySQL**：8.0 或更高版本
- **IDE**：IntelliJ IDEA / Eclipse / VS Code

### 1.2 生产环境
- **JDK**：17 或更高版本
- **MySQL**：8.0 或更高版本
- **操作系统**：Linux / Windows Server
- **Web服务器**：Nginx（可选，用于反向代理）

## 2. 开发环境部署

### 2.1 克隆项目
```bash
git clone <项目地址>
cd stray-animal-adoption
```

### 2.2 配置数据库

1. 创建数据库：
```sql
CREATE DATABASE stray_animal_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

2. 修改配置文件 `src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/stray_animal_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: your_username
    password: your_password
```

### 2.3 运行项目

**方式一：使用 Maven**
```bash
$env:JAVA_HOME="E:\ljcode\jdk"
$env:Path="$env:JAVA_HOME\bin;E:\ljcode\apache-maven-3.9.12\bin;$env:Path"
mvn clean install
mvn spring-boot:run
```

**方式二：使用 IDE**
1. 打开项目
2. 找到 `StrayAnimalApplication.java`
3. 右键运行 `Run 'StrayAnimalApplication'`

**方式三：打包后运行**
```bash
mvn clean package
java -jar target/stray-animal-adoption-1.0.0.jar
```

### 2.4 验证部署

1. 启动后在浏览器访问：`http://localhost:8080`  
   - 应自动跳转到登录页 `login.html`。
2. 使用数据库中已有用户登录（例如：`adopter1`）：
   - 登录成功后会跳转到前台首页 `index.html`。
3. 确认：
   - “动物列表”能正常加载；
   - 爱心捐赠表单可提交；
   - “我的申请”能在输入用户ID后查询到记录（如有）。

## 3. 生产环境部署

### 3.1 打包应用

```bash
mvn clean package -DskipTests
```

打包后的 jar 文件位于：`target/stray-animal-adoption-1.0.0.jar`

### 3.2 配置文件

创建生产环境配置文件 `application-prod.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://your-db-host:3306/stray_animal_db?useUnicode=true&characterEncoding=utf8&useSSL=true&serverTimezone=Asia/Shanghai
    username: your_prod_username
    password: your_prod_password
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
  
  jpa:
    hibernate:
      ddl-auto: validate  # 生产环境使用 validate，不自动建表
    show-sql: false
    properties:
      hibernate:
        format_sql: false

server:
  port: 8080
  servlet:
    context-path: /api

logging:
  level:
    root: INFO
    com.example.strayanimal: INFO
  file:
    name: /var/log/stray-animal/app.log
    max-size: 10MB
    max-history: 30
```

### 3.3 运行应用

**方式一：直接运行**
```bash
java -jar -Dspring.profiles.active=prod stray-animal-adoption-1.0.0.jar
```

**方式二：使用 systemd（Linux）**

创建服务文件 `/etc/systemd/system/stray-animal.service`：

```ini
[Unit]
Description=Stray Animal Adoption System
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/stray-animal
ExecStart=/usr/bin/java -jar -Dspring.profiles.active=prod /opt/stray-animal/stray-animal-adoption-1.0.0.jar
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable stray-animal
sudo systemctl start stray-animal
sudo systemctl status stray-animal
```

### 3.4 Nginx 反向代理配置（可选）

创建 Nginx 配置文件 `/etc/nginx/sites-available/stray-animal`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 重定向到 HTTPS（可选）
    # return 301 https://$server_name$request_uri;

    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源（如果有前端）
    location / {
        root /var/www/stray-animal-frontend;
        try_files $uri $uri/ /index.html;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/stray-animal /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3.5 HTTPS 配置（推荐）

使用 Let's Encrypt 免费证书：

```bash

sudo certbot --nginx -d your-domain.com
```

## 6. 性能优化建议

### 6.1 JVM 参数调优

```bash
java -jar -Xms512m -Xmx1024m \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  -Dspring.profiles.active=prod \
  stray-animal-adoption-1.0.0.jar
```

### 6.2 数据库优化

1. **连接池配置**：根据实际并发调整 `maximum-pool-size`
2. **索引优化**：确保常用查询字段有索引
3. **查询优化**：避免 N+1 查询问题，使用 `@EntityGraph` 或 `JOIN FETCH`

### 6.3 缓存策略

对于热点数据（如动物列表），建议使用 Redis 缓存：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

## 8. 常见问题

### 8.1 端口被占用

```bash
# 查找占用端口的进程
lsof -i :8080
# 或
netstat -tulpn | grep 8080

# 杀死进程
kill -9 <PID>
```

### 8.2 数据库连接失败

- 检查数据库服务是否启动
- 检查用户名密码是否正确
- 检查防火墙是否开放 3306 端口
- 检查 MySQL 是否允许远程连接

### 8.3 内存不足

调整 JVM 堆内存大小：
```bash
java -jar -Xms1g -Xmx2g stray-animal-adoption-1.0.0.jar
```

## 9. 安全建议

1. **生产环境关闭自动建表**：设置 `spring.jpa.hibernate.ddl-auto=validate`
2. **使用强密码**：数据库密码、管理员密码
3. **启用 HTTPS**：使用 SSL/TLS 加密传输
4. **配置防火墙**：只开放必要端口
5. **定期更新依赖**：修复安全漏洞
6. **实现认证授权**：添加 JWT 或 Session 认证
7. **输入验证**：防止 SQL 注入、XSS 攻击
8. **日志脱敏**：敏感信息不要记录到日志
