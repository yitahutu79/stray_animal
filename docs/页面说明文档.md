# 页面说明文档（前端视图）

## 1. 登录页：`login.html`

- **定位**：系统统一入口。
- **主要元素**：
  - 用户名输入框（示例：`adopter1` / `admin`）
  - 登录身份选择下拉框（自动识别 / 领养者 / 管理员）
  - “登录”按钮
- **交互逻辑**：
  - 点击登录时调用 `POST /api/auth/login`，传入 `username`；
  - 根据返回的 `userId`、`username`、`userType` 保存到 `localStorage`；
  - 若角色为领养者或选择“领养者” → 跳转 `index.html`；
  - 若角色为管理员/工作人员或选择“管理员” → 跳转 `stats.html`。

---

## 2. 领养者前台页：`index.html`

- **定位**：领养者主要操作界面。
- **模块划分**：
  1. 顶部导航栏：
     - “动物列表”、“爱心捐赠”、“我的申请”三个锚点；
     - 显示当前用户信息：“欢迎，用户名（角色）”；
     - “退出登录”按钮。
  2. 动物列表区域：
     - 按卡片形式展示所有动物；
     - 顶部下拉框按种类（狗/猫/其他）筛选；
     - 每张卡片包含名称、种类、品种、年龄、健康状况、简要描述；
     - 状态徽章：待领养（绿色）、领养中（黄色）、已领养（灰）、特别照顾（红）；
     - 如果状态为“待领养”，显示“申请领养”按钮；否则显示禁用按钮，防止二次申请。
  3. 爱心捐赠区域：
     - 左侧为捐赠表单（捐赠人姓名、类型、金额、备注、是否公开）；
     - 右侧为捐赠公示列表，展示近期公开捐赠及时间、状态。
  4. 我的申请区域：
     - “当前用户ID”输入框：默认从 `localStorage` 填充登录用户ID；
     - “刷新”按钮：调用 `GET /api/adoptions/my` 加载当前用户所有申请；
     - 使用表格展示：申请ID、动物名称、状态、评分、提交时间。

---

## 3. 申请审核页：`admin.html`

- **定位**：救助站管理员/工作人员审核领养申请的主界面。
- **主要功能**：
  - 顶部选择申请状态（SUBMITTED / UNDER_REVIEW / APPROVED / REJECTED）；
  - 表格展示该状态下所有申请，字段包括：
    - 申请ID、申请人ID、申请人姓名；
    - 动物ID、动物名称；
    - 系统评分；
    - 系统评价（对评分的文字解释）；
    - 申请理由、居住环境、有无其他宠物、收入水平、养宠经验；
    - 提交时间；
    - 操作按钮：通过 / 驳回。
- **交互逻辑**：
  - “刷新”按钮：调用 `GET /api/adoptions/admin?status=...`；
  - 点击“通过”：调用 `PUT /api/adoptions/{id}/status?status=APPROVED`；
    - 系统同时将对应动物状态改为“已领养”；
  - 点击“驳回”：弹出输入框录入驳回原因，调用带 `rejectReason` 的接口；
  - “退出登录”按钮：清空本地登录信息，返回登录页。

---

## 4. 宠物管理页：`pet-admin.html`

- **定位**：录入和维护救助站内动物信息。
- **布局结构**：
  - 左侧：新增 / 编辑动物档案表单；
  - 右侧：动物列表（可按状态过滤）。
- **表单字段**：
  - 名称、种类、品种、性别、年龄、体重、颜色、健康状况；
  - 免疫情况（复选框）、绝育情况（复选框）；
  - 性格特点；
  - 状态（待领养 / 领养中 / 已领养 / 特别照顾）；
  - 详细描述。
- **列表操作**：
  - “编辑”：将选中动物信息回填到左侧表单；
  - “删除”：调用 `DELETE /api/animals/{id}` 删除记录。

---

## 5. 捐赠管理页：`donation-admin.html`

- **定位**：统一查看和管理捐赠记录。
- **展示字段**：
  - 捐赠ID、捐赠人姓名、类型（资金/物资）、金额、状态、是否公开、备注、时间。
- **操作按钮**：
  - “确认”：将状态设置为 `CONFIRMED`；
  - “标记已使用”：将状态设置为 `USED`。
- **备注**：
  - 页面当前只管理主记录；捐赠使用去向通过 `/api/donations/{id}/usages` 接口可在后续扩展成弹窗或详情页面。

---

## 6. 统计分析页：`stats.html`

- **定位**：管理端的仪表盘首页。
- **统计卡片**：
  - **动物统计**：动物总数、待领养数、已领养数；
  - **申请统计**：领养申请总数、已通过数；
  - **捐赠统计**：捐赠记录总数、资金捐赠总额（仅统计 type=MONEY）。
- **导航**：
  - 顶部提供进入 `admin.html`、`pet-admin.html`、`donation-admin.html` 的快捷入口；
  - 支持退出登录。

---

## 7. 页面之间的导航关系

- `login.html`：统一入口 → 根据角色跳转到 `index.html` 或 `stats.html`；
- `index.html`：
  - 面向领养者，包含浏览、申请、捐赠、我的申请；
  - 可通过浏览器地址栏直接访问，但推荐从登录页进入以便带上用户信息。
- `stats.html`：
  - 面向管理员 / 工作人员的首页；
  - 提供到 `admin.html`（审核）、`pet-admin.html`（宠物）、`donation-admin.html`（捐赠）的导航。
- 所有管理端页面和前台页面右上角都提供统一的“退出登录”入口。
