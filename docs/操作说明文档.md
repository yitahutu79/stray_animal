# 操作说明文档（使用手册）

## 1. 登录与角色选择

### 1.1 访问入口

- 启动应用后，在浏览器访问：`http://localhost:8080`
- 系统会自动跳转到登录页：`login.html`

### 1.2 登录操作

1. 在登录页输入用户名：
   - 领养者示例：`adopter1`
   - 管理员示例：`admin`（需在数据库中配置 user_type=ADMIN）
2. 选择登录身份：
   - **自动识别**：根据数据库中的 `user_type` 判断角色；
   - **领养者**：强制按领养者身份进入前台；
   - **管理员 / 工作人员**：强制进入管理端首页。
3. 点击“登录”按钮：
   - 成功后，浏览器会根据角色自动跳转：
     - 领养者 → `index.html`
     - 管理端 → `stats.html`

> 说明：当前版本密码未做校验，后续可在此基础上接入真正的认证授权。

## 2. 领养者操作流程

### 2.1 浏览动物列表与状态

1. 登录为领养者后进入 `index.html`。
2. 页面顶部“动物列表”区域展示所有动物：
   - 每张卡片显示：名称、种类、品种、年龄、健康状况、简要描述；
   - 带有状态徽章：**待领养 / 领养中 / 已领养 / 特别照顾**；
   - 右上角下拉框可按种类（狗 / 猫 / 其他）进行筛选。
3. 状态说明：
   - **待领养**：按钮为“申请领养”，可点击提交申请；
   - **已领养 / 特别照顾**：按钮变为灰色并禁用，防止二次领养。

### 2.2 提交领养申请

1. 在“动物列表”中选择一只“待领养”的动物，点击“申请领养”按钮。
2. 弹出申请表单，填写信息：
   - 申请理由（必填）
   - 居住环境（如：小区住宅/自建房/合租等）
   - 是否有其他宠物
   - 收入水平（低 / 中 / 高）
   - 养宠经验描述
3. 在页面下方“我的领养申请（示例）”区域，确保“当前用户ID”已自动填入（登录后会自动带出，如为空可手动填自己的用户ID）。
4. 点击“提交申请”，成功后会提示“申请提交成功”。

### 2.3 查看我的申请记录

1. 在 `index.html` 下方“我的领养申请（示例）”区域：
   - 确认“当前用户ID”为当前登录用户；
   - 点击“刷新”按钮。
2. 列表中会显示：
   - 申请ID、动物名称、状态、评分、提交时间。
3. 状态说明：
   - `SUBMITTED`：已提交，待审核；
   - `APPROVED`：已通过，动物会被标记为“已领养”；
   - `REJECTED`：已驳回，可在后台备注中查看原因（仅工作人员可见）。

### 2.4 提交爱心捐赠

1. 在 `index.html` 中切换到“爱心捐赠”区域。
2. 在左侧表单中填写：
   - 捐赠人姓名（可选，可匿名）
   - 捐赠类型：资金捐赠 / 物资捐赠
   - 金额（仅资金捐赠时必填）
   - 备注说明
   - 是否同意公开捐赠信息
3. 点击“提交捐赠”，成功后右侧“捐赠公示”区域将显示该记录（若设置为公开）。

## 3. 管理员 / 工作人员操作流程

### 3.1 进入管理端首页（数据统计）

1. 使用管理员/工作人员账号在登录页登录；
2. 登录成功后进入 `stats.html`（数据统计分析页面）；
3. 可以看到：
   - 动物总数、待领养数、已领养数；
   - 领养申请总数、已通过数；
   - 捐赠总条数、资金捐赠总额。
4. 顶部导航可以跳转到：
   - 申请审核：`admin.html`
   - 宠物管理：`pet-admin.html`
   - 捐赠管理：`donation-admin.html`

### 3.2 领养申请审核（admin.html）

1. 在 `stats.html` 或其他管理页点击“申请审核”进入 `admin.html`。
2. 顶部选择要查看的申请状态（例如：`SUBMITTED` 待审核）。
3. 点击“刷新”加载列表。
4. 列表中每条申请包含：
   - 申请ID、申请人ID与姓名；
   - 动物ID与名称；
   - 系统评分与系统评价（文字说明评分依据）；
   - 申请理由、居住环境、有无其他宠物、收入水平、养宠经验；
   - 提交时间。
5. 审核操作：
   - 点击“通过”：将申请状态更新为 `APPROVED`，同时自动把对应动物状态更新为“已领养”，防止二次领养；
   - 点击“驳回”：输入驳回原因，状态变为 `REJECTED`。

### 3.3 宠物信息管理（pet-admin.html）

1. 在管理端导航中点击“宠物管理”进入 `pet-admin.html`。
2. 左侧表单：
   - 用于新增或编辑宠物档案；
   - 可填写名称、种类、品种、性别、年龄、体重、健康状况、免疫/绝育、性格特征、状态、详细描述。
3. 右侧列表：
   - 显示当前所有动物，支持按状态筛选；
   - 点击“编辑”：将该动物信息加载到左侧表单；
   - 点击“删除”：彻底删除该动物档案（请谨慎使用）。

### 3.4 捐赠记录管理（donation-admin.html）

1. 在管理端导航中点击“捐赠管理”进入 `donation-admin.html`。
2. 表格列出所有捐赠记录：
   - 包括捐赠人、类型（资金/物资）、金额、状态（PENDING/CONFIRMED/USED/PARTLY_USED）、是否公开、备注、时间。
3. 对每条记录可以执行：
   - “确认”：将状态更新为 `CONFIRMED`；
   - “标记已使用”：将状态更新为 `USED`。
4. 如需详细记录捐赠使用去向，可通过接口 `/api/donations/{id}/usages` 进一步扩展页面（当前版本为接口级支持）。

### 3.5 重新计算历史评分（可选运维操作）

当修改了评分规则后，可通过接口重新计算所有历史申请的分数：

- 浏览器或 Postman 访问：  
  `GET /api/adoptions/admin/recalculate-score`
- 执行成功返回 204，无正文；  
  之后在 `admin.html` 中刷新列表即可看到最新评分与系统评价。

## 4. 退出登录

在前台和所有管理端页面的右上角，都提供了“退出登录”按钮：

- 点击后会清除本地存储的 `currentUserId`、`currentUsername`、`currentUserType`；
- 并跳转回登录页 `login.html`。

## 5. 常见问题

1. **登录后“我的申请”查不到数据？**
   - 请确认“当前用户ID”输入框中的值与登录用户在 `user` 表中的 `id` 一致；
   - 提交过申请后再点击“刷新”。
2. **某只动物已经被领养，却还能看到“申请领养”按钮？**
   - 刷新页面查看最新状态；
   - 管理员审核通过后，系统会自动将该动物标记为“已领养”，前台按钮会变为灰色并禁用。
3. **捐赠金额统计不正确？**
   - 只有类型为 `MONEY` 的记录才会计入资金总额；
   - 请确认该捐赠记录的 `type` 是否为 `MONEY`，且 `amount` 已正确填写。
