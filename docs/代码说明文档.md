# 代码说明文档

本文从“包结构 → 关键类 → 核心业务流程”三个层次，对系统代码进行说明，便于阅读与二次开发。

---

## 1. 包结构总览

根包：`com.example.strayanimal`

### 1.1 启动与通用部分

- `StrayAnimalApplication`：Spring Boot 启动类。
- `common.BaseEntity`：
  - 作为所有实体的父类；
  - 提供 `createTime`、`updateTime` 自动时间戳字段（通过 Hibernate 注解维护）。

### 1.2 用户与登录模块

- 包：`user`
  - `User`：用户实体，字段包括 `username`、`password`、`userType`（ADMIN/STAFF/VOLUNTEER/ADOPTER）等。
  - `UserRepository`：JPA 仓库，支持 `findByUsername` 等方法。

- 包：`auth`
  - `AuthController`：
    - 路径前缀：`/api/auth`
    - 核心接口：`POST /login`，根据 `username` 查询用户并返回 `userId`、`username`、`userType`。
    - 当前版本不做密码校验，仅用于演示登录与角色分流。

- 包：`web`
  - `HomeController`：
    - 映射 `GET /`，重定向到 `login.html`，保证用户从登录页统一进入系统。

### 1.3 安全与跨域配置

- 包：`config`
  - `SecurityConfig`：
    - 关闭 CSRF；
    - 开启全局 CORS（允许前端通过浏览器调用接口）；
    - 将 `/api/**` 设置为全部放行（演示环境），生产环境可在此基础上增加认证授权逻辑。

## 2. 动物档案模块（animal）

- `Animal`：动物实体，对应表 `animal`，关键字段：
  - `name`、`species`（种类）、`breed`、`gender`、`age`、`weight`、`color`；
  - `healthStatus`、`vaccinated`、`neutered`、`characterFeature`；
  - `status`：待领养 / 领养中 / 已领养 / 特别照顾；
  - `description`：详细描述。

- `AnimalRepository`：
  - 继承 `JpaRepository<Animal, Long>`；
  - 自定义方法：
    - `Page<Animal> findByStatus(String status, Pageable pageable)`；
    - `Page<Animal> findByStatusAndSpecies(String status, String species, Pageable pageable)`；
    - `long countByStatus(String status)`：用于统计。

- `AnimalService`：
  - 提供分页查询服务 `list(status, species, page, size)`；
  - 提供单条查询、保存、删除等方法。

- `AnimalController`（`/api/animals`）：
  - `GET /api/animals`：分页查询，支持按状态和种类过滤；
  - `GET /api/animals/{id}`：详情；
  - `POST /api/animals`：新增；
  - `PUT /api/animals/{id}`：更新；
  - `DELETE /api/animals/{id}`：删除。

**前端对应页面**：`index.html`、`pet-admin.html`  
**关联逻辑**：在领养申请通过后，由 `AdoptionApplicationService.updateStatus` 将对应动物状态更新为“已领养”，前台据此禁用再次申请。

## 3. 领养申请与回访模块（adoption）

### 3.1 领养申请实体与仓库

- `AdoptionApplication`：
  - 关联字段：
    - `User applicant`（申请人，多对一）；
    - `Animal animal`（申请的动物，多对一）。
  - 业务字段：
    - `status`：SUBMITTED / UNDER_REVIEW / APPROVED / REJECTED / CANCELLED；
    - `reason`：申请理由；
    - `homeEnv`：居住环境描述；
    - `hasOtherPet`：是否有其他宠物；
    - `incomeLevel`：收入水平（LOW/MIDDLE/HIGH）；
    - `experience`：养宠经验；
    - `score`：系统计算的匹配评分（0–100）；
    - `rejectReason`：驳回原因。

- `AdoptionApplicationRepository`：
  - `Page<AdoptionApplication> findByApplicant_Id(Long applicantId, Pageable pageable)`；
  - `Page<AdoptionApplication> findByStatusOrderByScoreDesc(String status, Pageable pageable)`；
  - `long countByStatus(String status)`：统计不同状态的申请数量。

### 3.2 DTO 与评分解释

- `AdoptionApplicationDTO`：
  - 面向“我的申请”视图，包含 `id`、`animalName`、`status`、`score`、`createTime`。

- `AdoptionAdminDTO`：
  - 面向后台审核视图，包含：
    - 申请人 / 动物基础信息；
    - `status`、`score`；
    - `reason`、`homeEnv`、`hasOtherPet`、`incomeLevel`、`experience`；
    - `evaluation`：系统生成的综合评价文字；
    - `createTime`。

- `AdoptionScoreCalculator`：
  - 封装评分规则，先根据条件得到原始分 `raw`（0–50），再线性映射到 0–100：
    - 无其他宠物：+10；有其他宠物：+5；
    - 收入 HIGH：+20；MIDDLE：+10；LOW/其他：+5；
    - 有养宠经验：+20；
    - 最终 `score = round(raw * 100 / 50)`。

- `AdoptionEvaluationUtil`：
  - 根据申请人信息 + 动物信息 + 评分生成一段文字评价，解释匹配度高低的原因：
    - 是否有经验；
    - 是否有其他宠物；
    - 收入水平；
  

### 3.3 Service 与 Controller

- `AdoptionApplicationService`：
  - `apply(applicantId, animalId, request)`：
    - 读取用户与动物信息；
    - 对动物状态进行校验（“已领养”或“特别照顾”的动物不可再次申请）；
    - 调用评分器生成 `score`，保存申请。
  - `listMyApplications(applicantId, page, size)`：
    - 返回分页 DTO，用于前台“我的申请”展示。
  - `listByStatusForAdmin(status, page, size)`：
    - 返回按状态与评分排序的申请列表 DTO，用于审核页。
  - `updateStatus(id, status, rejectReason)`：
    - 更新申请状态；
    - 若状态为 `APPROVED`，同时将对应动物状态更新为“已领养”。
  - `recalculateAllScores()`：
    - 使用当前评分规则，为表中所有申请重新计算 `score`，用于评分规则变更后的批量更新。

- `AdoptionApplicationController`（`/api/adoptions`）：
  - `POST /apply`：提交申请；
  - `GET /my`：领养者查询自己的申请；
  - `GET /admin`：后台按状态分页查询申请；
  - `PUT /{id}/status`：审核通过/驳回；
  - `GET/POST /admin/recalculate-score`：批量重算历史评分。

### 3.4 回访模块

- 实体与仓库：
  - `AdoptionFollowup`：对应回访记录，关联 `AdoptionApplication` 与 `User staff`。
  - `AdoptionFollowupRepository`：按申请ID分页查询回访记录。

- Service 与 Controller：
  - `AdoptionFollowupService.addFollowup(applicationId, staffId, req)`：新增回访；
  - `AdoptionFollowupService.listByApplication(applicationId, ...)`：按申请查看所有回访；
  - `AdoptionFollowupController` 提供对应 REST 接口。

## 4. 捐赠与使用去向模块（donation）

- `Donation`：捐赠主记录，字段包括：
  - `donor`（可为空）与 `donorName`；
  - `type`：MONEY / ITEM；
  - `amount`：资金金额；
  - `status`：PENDING / CONFIRMED / USED / PARTLY_USED；
  - `publicFlag`：是否公开；
  - `remark`：备注。

- `DonationUsage`：捐赠使用记录，与 `Donation` 多对一关联，可选关联 `Animal`。

- `DonationRepository`：
  - `Page<Donation> findByPublicFlagTrue(Pageable pageable)`：公开捐赠列表；
  - `Double sumAllMoney()`：统计所有资金捐赠金额。

- `DonationUsageRepository`：
  - `Page<DonationUsage> findByDonation_Id(Long donationId, Pageable pageable)`。

- `DonationService`：
  - `createDonation(donorId, req)`：创建捐赠记录；
  - `listPublic(page, size)`：公开列表；
  - `listAdmin(page, size)`：后台分页查看所有捐赠；
  - `updateStatus(id, status)`：更新状态；
  - `addUsage(donationId, req)` / `listUsageByDonation(donationId, ...)`：使用去向记录管理。

- `DonationController`（`/api/donations`）：
  - `POST /`、`GET /public`、`GET /admin`、`PUT /{id}/status`；
  - `POST /{donationId}/usages`、`GET /{donationId}/usages`。

## 5. 统计模块（stats）

- `DashboardStats`：统计结果 DTO，包含：
  - `totalAnimals`、`animalsWaiting`、`animalsAdopted`；
  - `totalApplications`、`approvedApplications`；
  - `totalDonations`、`totalMoneyAmount`。

- `StatsService`：
  - 通过 `AnimalRepository`、`AdoptionApplicationRepository`、`DonationRepository` 进行聚合统计。

- `StatsController`：
  - `GET /api/admin/stats/overview`：统计分析接口，供 `stats.html` 使用。

---

## 6. 前端与后端的交互关系（简要）

- `login.html` → `AuthController`：登录获取用户信息与角色。
- `index.html`：
  - 动物列表 → `GET /api/animals`；
  - 申请领养 → `POST /api/adoptions/apply`；
  - 我的申请 → `GET /api/adoptions/my`；
  - 捐赠提交 → `POST /api/donations`；
  - 捐赠公示 → `GET /api/donations/public`。
- `admin.html`：
  - 审核列表 → `GET /api/adoptions/admin`；
  - 审核通过/驳回 → `PUT /api/adoptions/{id}/status`。
- `pet-admin.html`：
  - 动物列表与增删改 → `/api/animals`。
- `donation-admin.html`：
  - 捐赠记录与状态管理 → `/api/donations/admin`、`PUT /api/donations/{id}/status`。
- `stats.html`：
  - 概览统计 → `GET /api/admin/stats/overview`。
